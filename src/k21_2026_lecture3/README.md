# k21_2026_lecture3

第3回講義のプログラム例です。教材サイトの大規模アクセスログ、20名分の受講者アンケート、講義改善計画を共通テーマにして、Agents SDK のツール利用、Hosted tools、Structured output、Handoff、Guardrails、Tracing、MCP Server 連携を順に扱います。

## プログラム例

- `example01.ts`
  - 概要: LLMだけで教材サイトの大規模アクセスログを集計する例です。
  - 学習のねらい: 大きな数値の集計では、LLMの回答をそのまま信頼せず、検算可能な仕組みが必要になることを確認します。
- `example02.ts`
  - 概要: 汎用の四則演算ツール `calc` をエージェントに渡して、同じアクセスログ集計を行う例です。
  - 学習のねらい: 数値計算をツールに委ねることで、LLM単体より安定した結果を得る分担を学びます。
- `example03.ts`
  - 概要: アクセスログ分析に特化した `compute_access_log_summary` ツールで集計する例です。
  - 学習のねらい: 汎用計算式ではなく、業務に合った入力契約を設計することで誤用を減らす方法を学びます。
- `example04.ts`
  - 概要: Tavily検索ツールを自作し、講義改善に必要な Agents SDK 機能を調査する例です。
  - 学習のねらい: 外部検索APIをツール化し、エージェントが最新情報を参照して回答する構成を学びます。
- `example05.ts`
  - 概要: 前例の検索部分を Hosted web search に置き換える例です。
  - 学習のねらい: 自作ツールとHosted toolを比較し、組み込みツールで実装量を減らせる場面を学びます。
- `example06.ts`
  - 概要: Hosted code interpreterで `survey.csv` の第3回講義アンケートを分析する例です。
  - 学習のねらい: 表形式データの集計や並べ替えをコード実行環境に任せ、自然文で分析結果を得る流れを学びます。
- `example07.ts`
  - 概要: Structured output と集計ツールを併用し、`survey.csv` の分析結果をオブジェクトとして受け取る例です。
  - 学習のねらい: LLMの出力を、後続処理しやすい型付きデータとして扱う方法を学びます。
- `example08.ts`
  - 概要: Handoffでアンケート分析担当と授業改善計画担当の専門エージェントに分ける例です。
  - 学習のねらい: 役割ごとにエージェントを分割し、依頼内容に応じて処理を委譲する設計を学びます。
- `example09.ts`
  - 概要: Input guardrail と Output guardrail を使い、講義支援エージェントの安全な境界を作る例です。
  - 学習のねらい: 不適切な入力やリスクのある出力を検出し、エージェントの実行を制御する方法を学びます。
- `example10.ts`
  - 概要: Tracingでツール利用を含む講義改善フローを記録する例です。
  - 学習のねらい: エージェントの実行過程を追跡し、デバッグや振り返りに使えるようにする方法を学びます。
- `example11.ts`
  - 概要: Excel MCP Serverを使い、講義改善データをExcelファイルとして作成・分析する例です。
  - 学習のねらい: MCP Serverを通じて外部アプリケーション形式のファイルを操作し、分析作業へつなげる方法を学びます。
- `example12.ts`
  - 概要: 検索、アクセスログ集計、アンケート集計、input guardrail、structured output、tracingを組み合わせた総合例です。
  - 学習のねらい: 第3回で扱う主要概念を統合し、実務に近い講義改善ワークフローとして構成する方法を学びます。

## 補足

- `example04.ts` は Tavily API キーが必要です。
- `survey.csv` は20名分の事前アンケートデータです。
- `example11.ts` は `survey.csv` をもとに `scores.xlsx` を操作する例です。
