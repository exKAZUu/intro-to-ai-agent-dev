# k21_2026_lecture3

第3回講義のプログラム例です。LLMだけに処理を任せた場合の不安定さを出発点に、Agents SDK のツール利用、Hosted tools、Structured output、Handoff、Guardrails、Tracing、MCP Server 連携を順に扱います。

## プログラム例

- `example01.ts`
  - 概要: LLMだけで講義アンケートの人数計算を行わせる例です。
  - 学習のねらい: LLMは自然文理解が得意でも、条件の読み落としや計算ミスが起きうることを確認します。
- `example02.ts`
  - 概要: 汎用の四則演算ツール `calc` をエージェントに渡して人数計算を行う例です。
  - 学習のねらい: 計算をツールに委ねることで、LLM単体より数値処理を安定させられることを学びます。
- `example03.ts`
  - 概要: 人数、グループ人数、グループ数を受け取るドメイン特化ツールで集計する例です。
  - 学習のねらい: 汎用ツールよりも、用途に合わせた入力契約を設計するほうが誤用を減らしやすいことを学びます。
- `example04.ts`
  - 概要: グループ計算を、TA数や教室数を含む演習運営計画へ拡張する例です。
  - 学習のねらい: ツールの責務をドメインの業務単位に広げ、LLMに説明と判断を担当させる分担を学びます。
- `example05.ts`
  - 概要: Tavily検索ツールを自作し、Agents SDK の tools、MCP、guardrails を調査する例です。
  - 学習のねらい: 外部APIをツール化し、エージェントが最新情報を参照して回答する構成を学びます。
- `example06.ts`
  - 概要: Tavily検索と授業時間計算ツールを組み合わせ、90分授業のローテーション計画を作る例です。
  - 学習のねらい: 複数ツールを組み合わせ、情報収集と決定的な計算を1つのエージェントで扱う方法を学びます。
- `example07.ts`
  - 概要: 題材確認の検索部分を Hosted web search に置き換える例です。
  - 学習のねらい: 自作ツールとHosted toolの違いを見比べ、組み込みツールで実装量を減らせる場面を学びます。
- `example08.ts`
  - 概要: Hosted code interpreterで試行授業後アンケートを分析する例です。
  - 学習のねらい: 表形式データの集計や並べ替えをコード実行環境に任せ、自然文で分析結果を得る流れを学びます。
- `example09.ts`
  - 概要: Structured output と集計ツールを併用し、アンケート分析結果をオブジェクトとして受け取る例です。
  - 学習のねらい: LLMの出力をプログラムから扱いやすい型付きデータにし、後続処理へつなげる方法を学びます。
- `example10.ts`
  - 概要: Handoffでアンケート分析担当と改善案作成担当の専門エージェントに分ける例です。
  - 学習のねらい: 役割ごとにエージェントを分割し、依頼内容に応じて処理を委譲する設計を学びます。
- `example11.ts`
  - 概要: Input guardrailで講義改善エージェントに不適切な依頼が届く前に停止する例です。
  - 学習のねらい: 入力段階で扱うべきでない依頼を検出し、エージェントの実行を制御する方法を学びます。
- `example12.ts`
  - 概要: Output guardrailで講義紹介文に不適切な保証表現が含まれた場合に停止する例です。
  - 学習のねらい: 生成後の出力を検査し、禁止表現やリスクのある回答を外へ出さない仕組みを学びます。
- `example13.ts`
  - 概要: Tracingで handoff とツール利用を含む講義改善フローを記録する例です。
  - 学習のねらい: エージェントの実行過程を追跡し、デバッグや振り返りに使えるようにする方法を学びます。
- `example14.ts`
  - 概要: Playwright MCP Serverを接続し、Agents SDKドキュメントをブラウザで確認する例です。
  - 学習のねらい: MCP Serverを通じて外部ツールをエージェントに接続し、ブラウザ操作を委譲する方法を学びます。
- `example15.ts`
  - 概要: Excel MCP Serverを使い、アンケートデータをExcelファイルに作成して分析する例です。
  - 学習のねらい: MCP経由で既存アプリケーション形式のファイルを操作し、分析作業へつなげる方法を学びます。
- `example16.ts`
  - 概要: 検索、集計、時間配分、input guardrail、structured output、tracingを組み合わせた総合例です。
  - 学習のねらい: 第3回で扱う主要概念を統合し、実務に近い講義改善ワークフローとして構成する方法を学びます。

## 補足

- `example05.ts` と `example06.ts` は Tavily API キーが必要です。
- `example14.ts` は Playwright MCP Server を利用するため、事前に `npx --yes playwright install chromium` を実行しておきます。
- `example15.ts` は `scores.xlsx` を操作する例です。
