# k21_2026_lecture3

ワークショップのプログラム例です。ある学習サイトの大規模アクセスログ、20名分の参加者アンケート、改善計画を共通テーマにして、Agents SDK のツール利用、Hosted tools、Structured output、Handoff、Guardrails、Tracing、MCP Server 連携を順に扱います。

## プログラム例

- `example01.ts`
  - 概要: LLMだけで学習サイトの大規模アクセスログを集計する例です。
  - 学習のねらい: 大きな数値の集計では、LLMの回答が正しく見える場合でも、期待値と照合する仕組みが必要になることを確認します。
- `example02.ts`
  - 概要: 汎用の四則演算ツール `calc` をエージェントに渡して、学習サイトのアクセスログ集計を行う例です。
  - 学習のねらい: 数値計算をツールに委ねることで、LLM単体より安定した結果を得る分担を学びます。
- `example03.ts`
  - 概要: アクセスログ分析に特化した `compute_access_log_summary` ツールで集計する例です。
  - 学習のねらい: 汎用計算式ではなく、業務に合った入力契約を設計することで誤用を減らす方法を学びます。
- `example04.ts`
  - 概要: Tavily検索ツールを自作し、改善計画に必要な Agents SDK 機能を調査する例です。
  - 学習のねらい: 外部検索APIをツール化し、公式情報に限定して根拠URLつきで回答する構成を学びます。
- `example05.ts`
  - 概要: Hosted web search を使い、改善計画に必要な Agents SDK 機能を調査する例です。
  - 学習のねらい: 組み込みの検索ツールで実装量を減らしながら、根拠URLを確認する方法を学びます。
- `example06.ts`
  - 概要: Hosted code interpreterで `survey.csv` の演習アンケートを分析する例です。
  - 学習のねらい: 表形式データの集計や並べ替えをコード実行環境に任せ、プログラム側の検算値と比較する流れを学びます。
- `example07.ts`
  - 概要: Structured output と集計ツールを併用し、`survey.csv` の分析結果をオブジェクトとして受け取る例です。
  - 学習のねらい: LLMの出力を、後続処理しやすい型付きデータとして扱い、主要値を検算する方法を学びます。
- `example08.ts`
  - 概要: Handoffでアンケート分析担当と改善計画担当の専門エージェントに分ける例です。
  - 学習のねらい: 役割ごとにエージェントを分割し、依頼内容に応じて処理を委譲する設計と委譲ログの見方を学びます。
- `example09.ts`
  - 概要: Input guardrail と Output guardrail を使い、学習支援エージェントの安全な境界を作る例です。
  - 学習のねらい: 不適切な入力やリスクのある出力を検出し、エージェントの実行を制御する方法を学びます。
- `example10.ts`
  - 概要: Tracingでツール利用を含む改善フローを記録する例です。
  - 学習のねらい: エージェントの実行過程を追跡し、Trace名とツール呼び出しをデバッグや振り返りに使う方法を学びます。
- `example11.ts`
  - 概要: Excel MCP Serverを使い、アンケートデータをExcelファイルとして作成・分析する例です。
  - 学習のねらい: MCP Serverを通じて外部アプリケーション形式のファイルを操作し、分析作業へつなげる方法を学びます。
- `example12.ts`
  - 概要: 検索、アクセスログ集計、アンケート集計、input guardrail、structured output、tracingを組み合わせた総合例です。
  - 学習のねらい: このディレクトリで扱う主要概念を統合し、公式情報の根拠URLも含む実務に近い改善ワークフローとして構成する方法を学びます。

## 補足

- `example04.ts` は Tavily API キーが必要です。
- `survey.csv` は20名分の事前アンケートデータです。
- `example11.ts` は `survey.csv` をもとに新しいExcelファイルを作成する例です。
